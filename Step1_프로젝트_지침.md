# Step 1 — 서울 한옥 에어비앤비 전수조사 프로젝트 지침

## 프로젝트 개요

에어비앤비에 등록된 서울의 한옥 숙소를 전수 조사하는 프로젝트입니다.
이 Cowork 창은 **Step 1 전용**입니다. Step 2(상세 데이터 수집)는 별도 창에서 진행합니다.

## Step 1이 하는 일

1. Playwright로 에어비앤비 브라우저 열기
2. 내부 API 키 자동 캡처
3. **서울 5개 구 × 80개 행정동**을 하나씩 검색
4. 각 동에서 **"한옥" 키워드** 포함 숙소만 필터링
5. 동마다 중간 저장 (중단 후 재실행 시 이어서 진행)
6. 최종 결과를 `한옥_목록.xlsx`로 저장

### 수집 대상 구·동
- **종로구** 17개 동 (북촌, 서촌, 익선동 포함)
- **중구** 15개 동 (명동, 을지로 인근)
- **성북구** 18개 동 (성북동 고택 밀집)
- **동대문구** 14개 동
- **서대문구** 16개 동

총 **80개 동**, 예상 소요 시간: **약 20~40분**

### 출력 컬럼
| 컬럼 | 설명 |
|------|------|
| 순위 | 후기 수 기준 정렬 |
| 구 | 행정구 |
| 동 | 행정동 |
| 숙소 이름 | 에어비앤비 등록 이름 |
| 링크 (URL) | 숙소 페이지 직접 링크 |
| 별점 | 평균 별점 |
| 후기 수 | 누적 후기 수 |
| 슈퍼호스트 | Y / 공백 |
| 게스트 픽 | Y / 공백 |

## 기술 참고 사항

### API 방식
에어비앤비 내부 GraphQL API (`StaysSearch`)를 직접 호출합니다.
- 브라우저에서 `x-airbnb-api-key` 헤더 자동 캡처
- `rawParams` 배열에서 `query` 필드를 동 이름으로 교체하여 지역 변경
- 날짜 파라미터 없이 검색 (최대 숙소 노출)

### 한옥 판별 로직
숙소 이름에 다음 키워드 중 하나라도 포함되면 한옥으로 분류:
```
한옥, hanok, 韓屋, 기와, 고택, 전통가옥, 전통 가옥, 한옥스테이, 한옥 스테이
```

### 페이지네이션
- 동당 최대 15페이지 × 18개 = 최대 270개 조회
- 응답의 `paginationInfo.pageCursors` 배열로 페이지 전환
